refinery by adamh

Raw image processing library

Every expensive digital camera has its own special RAW format, the most unrefined and detailed format it can use to store photographs. This is useful because RAW files lose less detail and add fewer imperfections than your camera's JPEGs. But at some point you'll want to view and manipulate these RAW files.

All these RAW formats share a few common traits:

  • They're complicated, so it's hard to write your own code to handle them.
  • They're unconventional, so it's hard to find somebody else's code to handle them.
  • They're big, so it's very hard to handle them quickly.

Refinery is a library programs can use to extract image data from RAW files. This project doesn't provide the best programs for converting from RAW format: it strives to be a fast, easy-to-use cog in other developers' great programs.

Here's what refinery does. (This list is detailed because refinery doesn't do much, nor is it meant to.)

  • Reads images from files or other standard-library input streams (e.g. memory buffers)
  • Interpolates colors where the image doesn't store them (this is important: your camera only captures one color per pixel, so RAW files only store one color per pixel)
  • Writes images in PPM format to files or other standard-library output streams (e.g. memory buffers)
  • Manipulates images in standard, basic ways (for instance, color conversions)

Dependencies

Runtime dependencies:

  • Exiv2 for reading image metadata: http://www.exiv2.org/

Additional compile-time dependencies:

  • googletest C++ unit testing framework: http://code.google.com/p/googletest/
  • Boost C++ libraries: http://www.boost.org/
  • CMake cross-platform build system: http://www.cmake.org/

Install

This project's Makefile is generated by CMake.

To regenerate the Makefile, run "cmake -D CMAKE_BUILD_TYPE=Release .".

To make a debug-suited Makefile, use "Debug" instead of "Release".

Finally, "make" and (if installing) "sudo make install".
